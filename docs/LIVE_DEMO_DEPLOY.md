# Live Demo Deployment (Render)

This project is now configured for one-click Render deployment via `render.yaml`.

## Prerequisites

- GitHub repository connected to Render
- Branch: `main`

## Steps

1. Open Render dashboard and choose **New +** -> **Blueprint**.
2. Select this repository: `tasheayansa6/diabetes-prediction-system`.
3. Render will detect `render.yaml` automatically.
4. Click **Apply** to create the web service.
5. Wait for build and deployment to complete.

## Default Runtime Config

- Build command: `pip install -r requirements_production.txt`
- Start command: `gunicorn run:app --workers 2 --threads 4 --timeout 120 --bind 0.0.0.0:$PORT`
- Environment:
  - `FLASK_ENV=production`
  - `SECRET_KEY` auto-generated by Render
  - `DATABASE_URL=sqlite:///database/diabetes.db` (demo only)

## Important Notes

- Free-tier demo uses sqlite for simplicity.
- For production-grade deployment, switch to managed PostgreSQL and set `DATABASE_URL`.
- After deploy, verify:
  - `/health`
  - `/api/model/info`
  - login/register flow
  - prediction submission flow
